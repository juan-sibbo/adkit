# OpenRTB Deal Object — Technical reference

## Deal object in OpenRTB 2.x (inside `pmp`)

```json
{
  "pmp": {
    "private_auction": 1,
    "deals": [
      {
        "id": "DEAL-ID-HERE",
        "at": 1,
        "bidfloor": 5.00,
        "bidfloorcur": "USD",
        "wseat": ["seat-id-buyer"],
        "wadomain": ["advertiser.com"],
        "ext": {}
      }
    ]
  }
}
```

---

## Deal object fields — mandatory and optional

| Field | Type | Mandatory | Description |
|---|---|---|---|
| `id` | string | **Yes** | Unique deal ID — must match exactly the one configured in the DSP |
| `at` | integer | Recommended | Auction type: `1` = first price, `2` = second price, `3` = fixed price (PG) |
| `bidfloor` | float | Recommended | Deal floor in CPM — buyers below this value are discarded |
| `bidfloorcur` | string | Recommended | Floor currency — defaults to USD if not specified |
| `wseat` | array[string] | Optional | Authorized buyer seat IDs — if empty, any buyer can bid |
| `wadomain` | array[string] | Optional | Authorized advertiser domains |
| `ext` | object | Optional | Proprietary SSP extensions |

**Critical field for PG:** `at=3` (fixed price) is mandatory in PG deals. If `at=1` or `at=2` arrives, the buyer can win below the agreed price.

---

## `private_auction` in the `pmp` object

| Value | Meaning |
|---|---|
| `0` | Open auction — the deal ID gives preferential access but inventory is also open |
| `1` | Private auction — only buyers with the configured deal ID can bid |

**Frequent anti-pattern:** setting `private_auction=1` on a PMP deal and not including the correct deal ID in `wseat`, which blocks all bids including those from the target buyer.

## Common extensions by SSP

### Google AdX / Open Bidding
Google does not use the standard `pmp.deals` field in Open Bidding — the deal is managed entirely from GAM through the yield group and PG/PD line item configuration. The deal ID does not travel in the standard bid request in the same way.

- In Open Bidding, deal targeting is configured in GAM (`gam-adops`)
- Low fill diagnosis: verify active yield group, PG line item with correct deal ID, authorized buyer seat in AdX

### Magnite / Rubicon
```json
"ext": {
  "rubicon": {
    "pmp": {
      "type": "pmp"  // or "pg" or "pd"
    }
  }
}
```
- Magnite exposes bid rate and win rate reporting per deal ID in its UI — use as first diagnostic source
- For PG, the `at=3` field is mandatory — verify in bid request logs
- Magnite supports `wseat` to restrict buyers — if the deal is not delivering, verify the buyer's seat is on the list
- Deal Health Report available in UI: impressions, bid rate, win rate, loss reasons by category

### Triplelift
- Triplelift uses proprietary alphanumeric deal IDs — format is case-sensitive
- In native deals, the deal object includes native format extensions in `ext.triplelift`
- Deal health reporting available in Triplelift UI — includes won impressions, bid rate, and loss reasons
- Mandatory creative review for native before the deal can deliver

### Xandr / Microsoft
- Xandr uses "deal" for PMP and "guaranteed deal" for PG
- Xandr deal ID is numeric (e.g., `12345678`)
- In bid requests: `pmp.deals[0].ext.appnexus` for proprietary extensions
- Diagnosis: Deal Monitor in Xandr UI — shows bid activity, win rate, and non-delivery reasons
- Seat ID for `wseat` in Xandr corresponds to the buyer's "member ID"

### Index Exchange (IX)
- IX uses deal IDs with "IX-" prefix — verify exact format with the account manager
- Deal health reporting available via Deal ID Report in UI or via API
- IX deal types: "Private Auction", "Preferred Deal", "Guaranteed"
- Low fill diagnosis: IX allows seeing bid rate and loss reasons at deal ID level

### PubMatic
- PubMatic deal IDs are numeric, generated by the platform
- PubMatic Deal Health Report in UI: impressions, bid rate, win rate, revenue
- For PG: `at=3` in the deal object — verify in logs
- PubMatic exposes a "Deal Troubleshooter" in the UI to diagnose non-delivering deals

### Criteo
- Criteo typically acts as a buyer (DSP), not as an SSP — deals with Criteo are PMPs where Criteo is the buyer
- The deal is configured in the primary SSP (Magnite, IX, etc.) with Criteo as the authorized buyer
- Criteo's seat ID in OpenRTB varies by SSP — get from the Criteo account manager
- Criteo has its own ad quality and brand safety restrictions that can block the deal even if correctly configured

### Teads
- Teads is primarily an outstream video and contextual display SSP
- Deals in Teads have an attention metrics component — buyers can filter by attention score
- If the deal is not delivering, verify if the buyer has configured a minimum attention threshold
- Deal IDs in Teads are alphanumeric — sensitive to exact format

### Richaudience / Adagio
- Richaudience: SSP specialized in outstream video. Deals are typically direct PMPs.
- Adagio: SSP focused on attention. Buyers can configure attention or viewability score filters that block fill.
- For both: verify the buyer does not have an attention or viewability threshold configured that the inventory does not meet

### The Trade Desk (TTD) — as buyer
TTD is the reference DSP. When TTD is the buyer and the deal is not delivering:
- Verify the deal is "Active" in TTD (not "Pending" or "Paused")
- TTD uses Merchant IDs to identify publishers — the SSP must have the correct Merchant ID
- TTD has strict ads.txt validation — a missing or incorrect SSP line completely blocks the deal
- TTD exposes a "Deal Health" report in its UI to see bid rate and loss reasons
- For PG: TTD calls PG deals "Guaranteed Contracts" — verify the deal type matches

## Deal types — operational differences

| Type | `at` | `private_auction` | Description | Main risk |
|---|---|---|---|---|
| **PMP / PA** (Private Marketplace / Private Auction) | `1` or `2` | `1` | Publisher offers preferential inventory to a group of buyers. They compete with each other and with the open market if `private_auction=0`. | Deal ID misconfigured in DSP; floor too high |
| **PD** (Preferred Deal) | `2` | `1` | Buyer has first look at inventory at a fixed price before the auction. No volume guarantee. | Buyer not bidding even with access; non-competitive floor |
| **PG** (Programmatic Guaranteed) | `3` | `1` | Guaranteed volume and price. The buyer must deliver the agreed volume. | Incorrect `at` in bid request; GAM line item not activated; creative not approved |

## Diagnosis by symptom — deal object fields

| Symptom | Field to verify | Verification |
|---|---|---|
| Deal ID not recognized by DSP | `id` | Compare character by character — spaces and case matter |
| Final CPM below floor | `at` + `bidfloor` | For PG, `at` must be `3` (fixed price) |
| Correct buyer cannot bid | `wseat` | Verify buyer's seat ID is on the list or that `wseat` is empty |
| Deal delivers with unexpected advertisers | `wadomain` | Empty field = any advertiser can win |
| PMP delivers in open auction when it should not | `private_auction` | Must be `1` for private auction |

## Bid request debugging — checklist

To verify the deal object reaches the buyer correctly:

1. Get a sample of bid requests from the SSP (debug log or SSP audit tool)
2. Confirm `pmp.deals[0].id` matches exactly the deal ID configured in the DSP
3. Confirm `at` according to deal type
4. Confirm `bidfloor` and `bidfloorcur`
5. If `wseat` is populated, confirm it includes the target buyer's seat
6. Verify `pmp.private_auction` has the expected value
7. Ask the buyer to confirm the deal ID is active in their DSP and the correct seat

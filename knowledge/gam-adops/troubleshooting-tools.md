# Native GAM Troubleshooting Tools

First operational resource before formulating hypotheses without data. Each tool has a specific scope — use the right one for the environment and symptom.

---

## Line Item Troubleshoot Tab

**Where:** inside the line item, "Troubleshoot" tab (or "Diagnose" in some interface versions).

**What it does:** simulates the ad serving process for a given ad unit and targeting, and indicates why the line item is not winning (or is winning) the auction in that context.

**Information returned:**
- Whether the line item is eligible for the queried ad unit
- Why it is not winning: lower priority than another line item, targeting not met, creative not available for that size, frequency cap reached
- Which line item is winning instead

**When to use it:** first resource when a line item is not delivering. Before manually reviewing targeting, creative sizes, or priorities.

**Limitation:** the simulation uses the context the user enters manually (ad unit, key-values, geo, device). If the context does not reflect real traffic, the result may be misleading. Verify that the key-values entered exactly match what the page sends.

## Google Publisher Console

**How to access:** add `?google_console=1` to the page URL, or open from the "Google Publisher Console" Chrome extension.

**What it does:** shows in real time, on the publisher's page, which ad is serving in each slot, which line item won, what targeting was applied, and what competing bids there were.

**Useful diagnostic information:**
- Winning line item and its priority
- Key-values that reached the ad server from that page
- Requested sizes vs served creative sizes
- Whether there are unfilled slots and why (no eligible buyers, all filtered, etc.)
- CPM of the winner and competitors (on programmatic inventory)

**When to use it:**
- Verify that key-values sent from the page match the line item targeting
- Confirm which line item is winning when another is not delivering
- Verify the real requested sizes of the slot

**Limitation:** requires access to the page in a browser. Does not work for app or CTV traffic. For video/audio, use the specific inspector.

## Webpage Ads Inspector (GAM Ads Inspector)

**How to access:** "Google Publisher Tag (GPT) Ad Inspector" Chrome extension, or opening the panel from the browser console with `googletag.openConsole()`.

**What it does:** browser panel showing the technical detail of each GPT slot on the page: slot definition, set targeting, auction status, served creative.

**Useful information:**
- Exact slot definition (ad unit code, declared sizes)
- Active page-level and slot-level targeting at the time of the auction
- Correlator status and ad server call status
- If the slot is in lazy loading: whether the call has fired or not
- GPT configuration errors (slots without display(), invalid sizes, etc.)

**Difference from Publisher Console:** Ads Inspector is more technical (GPT/JS level); Publisher Console is more operational (what served and why from GAM's perspective).

**When to use it:** when the symptom is technical — the slot is not making the ad server call, key-values are not being set, there are GPT errors. For auction result analysis, prefer Publisher Console.

## Video/Audio Ads Inspector

**How to access:** tool within GAM (Admin > Video/Audio Inspector, or from the video line item).

**What it does:** allows entering the parameters of a video/audio ad request and simulating which ad GAM would serve, including the VAST response.

**Useful information:**
- Test ad tag with the configured parameters
- VAST response generated by GAM for that request
- Which line item wins and why
- Configuration errors (sizes, duration, formats)

**Important limitations:**
- Does not support context-dependent features well such as session-based frequency caps or session video ad rules `[360]` — the simulation cannot replicate the real user session state
- For VAST beacon debugging, quartile tracking, or player errors → `video-adtech`

**When to use it:** verify that a video ad tag generates the correct VAST before sending it to a client or to a player. First resource for "the video ad tag returns no ad".

## Operational Diagnostic Reports in GAM

GAM reports are themselves a troubleshooting tool. Key dimensions and metrics for diagnosis:

### For line item delivery issues

| Dimension / Metric | What it reveals |
|---|---|
| **Ad unit** | Break down delivery by position — isolate which specific ad unit has the problem |
| **Line item** | Compare actual delivery vs goal |
| **Creative** | Identify if a specific creative has 0 fill (rejected, incorrect size) |
| **Requested ad sizes** | Sizes the slot is actually requesting — compare with creative sizes |
| **Matched requests** | Ad requests that found at least one eligible line item — if low, the problem is in targeting/priority |
| **Unmatched requests** | Ad requests with no eligible line item — indicates monetization gaps |
| **% Impressions / Matched requests** | Fill rate within eligible requests — if low with high matched requests, there is competition or filtering |

### For programmatic issues

| Dimension / Metric | What it reveals |
|---|---|
| **Programmatic channel** | Break down by Open Auction, PD, PA, PG, Open Bidding |
| **Deal ID** | Delivery and revenue by specific deal |
| **Buyer network** | Which DSPs or networks are buying (or not buying) |
| **Pricing rule** | Which floor was applied — if the floor is too high, it will block bids |
| **Yield group** | Which yield group intermediated — if there are ad units without a yield group, there is no Open Bidding on them |
| **Ad request CPM** | Estimated inventory CPM — useful for calibrating floors |

### For discrepancies

See `discrepancy.md` for the full protocol. Key metrics in this context:

| Metric | What it reveals |
|---|---|
| **Impressions** | Impressions counted by GAM (IVT filtered) |
| **Total impressions (unfiltered)** `[360]` | Impressions before IVT filtering — compare with SSP metrics |
| **Invalid impressions** `[360]` | Volume of IVT filtered — may explain part of the discrepancy with SSPs |
| **Revenue** | Net revenue reported by GAM |

## When to escalate beyond native tools

Native GAM tools have limits. Escalate to Google support or the SSP when:

- The Troubleshoot tab indicates the line item should win but there are no impressions in the report
- Publisher Console shows the line item as winner but the creative does not render (technical issue with the creative or tag)
- The discrepancy persists after exhausting the `discrepancy.md` protocol
- There is ad serving behavior that does not correspond to any visible configuration (possible GAM bug)

Always document before escalating: tool used, parameters entered, result obtained, period, and affected ad units.
